# 프로젝트 에이전트 및 규칙 (Project Agents & Rules)
1. github 스케줄을 이용해서 gumdrop.tistory.com에 자동을 글을 발행한다.
2. 글은 1에 1개씩 구글 SEO 최적화된 글 쓴다.
3. 글은 NVIDIA NIM api를 이용해서 한글로 6000자 이상의 글을 쓴다.
4. 글은 최대한 구글 검색에 노출될 수 있는 키워드를 많이 사용해서 작성한다. 
5. 글의 제목도 키워드를 많이 사용해서 작성한다. 
6. 이미지도 SEO에 잘되도록 ai를 이용해서 생성해서 작성한다. 
7. 크롤러로 키워드를 매일 확인해서 글감이 없다면 바로 크롤러를 이용해서 구글 트렌드 키워드를 가져와 글을 작성한다.
8. 크롤러 api 호출시 timeout 에러에 항상 대비한다.

## 프로젝트 개요 (Project Context)
- **프로젝트명**: NEW AUTOBLOG
- **주소**: https://gumdrop.tistory.com

## 개발 가이드라인 (Development Guidelines)
1. **SEO 최적화**: 검색 엔진 최적화를 위해 메타 태그, `robots.txt`, `sitemap.xml`이 올바르게 유지되도록 신경 씁니다.
2. **애드센스(AdSense)**: 레이아웃이나 페이지를 수정할 때 기존에 삽입된 Google 애드센스 관련 스크립트가 누락되지 않도록 주의합니다.
3. **마크다운(Markdown)**: 블로그 포스트 작성 시 GFM(GitHub Flavored Markdown) 형식을 사용합니다.
4. **배포 확인**: github를 사용하고 있으니 배포 환경을 확인하고 배포 테스트 한다.
5. **오류에 대해서 항상 미리 확인**: 검색어 확인시 Failed to extract Google realtime keyword에 대해서 대비한다. 
6. **에러에 대비**: TimeoutError 30초에 항상 대비한다. 에러가 예상이 되면 예상되는 대비를 끝까지 수정한다. 필요없는 코드 삭제해서 에러는 만들지 않는다. 
7. **테스트 철저**: 수정 후, 테스트를 철저히 한다.
8. **소스 수정시 필요한 부분만 수정**: 존재하지 않는 함수 이름을 호출하지 않도록 작성한다. 
- 파일을 수정할 때는 반드시, 언제든지 필요한 속성, 필요한 부분만 수정한다
- Playwright는 특정 상황에서 url 대신 domain과 path의 조합을 명시적으로 요구하는 깐깐한 검증 로직를 가지고 있다.
9. **자동 발행** : 자동 발행되는 글은 반드시 한글로만 되도록 한다. 
- 발행된 글에서는 절대 마크다운 문법 기호가 안보이도록 한다.
- 정치적 키워드 및 정치와 관련된 키워드, 내용은 작성, 발행하지 않는다
10. **이미지 삽입 방식**: 블로그 페이지에 이미지는 마크다운 형식의 url로 표시한다. 절대 /assets/images/ 폴더에 이미지를 업로드 하지 않는다.
